此README文件为一次代码复盘，分析自己在编写代码时遇到的问题。

作业的代码要求是用自己熟悉的程序设计语言实现对一个命题公式的：

（1）真值表的打印
（2）公式类型的判断
（3）两种主范式的求解

抛开代码运行环境这个繁琐、暧昧的问题，我在编程过程中遇到的最困难的事就是把问题拆解成几个部分来实现。这件事想起来简单，做起来却因拆解问题的思路复杂而变得困难。

例如，在思考如何实现第一个问题时，我只考虑了打(真值)表，仅仅将每种变元的取值呈现在输出结果上面，而变元取值无法在后续求解问题时使用，且打表还花费了我很多时间，实属不明智了。

此外，我遇到的问题还有很多：

1.把每一个命题公式的计算想得太简单了，没有考虑到设计公式类型的优化转换(中缀表达式(如p&q)转后缀表达式(如pq&))、公式去空格和计算公式真值的算法，导致花费了较多不必要的时间。

2.对一些临界条件进行了重复判断，比如，判断变元是否是小写字母，本来仅仅 小写&&字母 即可结束判断，我却添加了 !小写-->false 的if判断，是否提升代码健壮性不说，至少写法是不够简洁的。

3.求解第三个问题时，由于自己对主范式的理解不够深刻，最初写代码时没有考虑求出极小/大项，本来思路就混乱，该部分的代码更是基本上写不出来。

4.在几个小时后，我写好了所需的类和函数，准备在main函数中调用。可是，我没有

其次，是分析要使用的数据结构。




提及本次作业的代码运行环境，我使用的是自以为最熟悉（仅仅是安装了许多插件，为IDE配置了一个图片背景，亲手过写代码的次数屈指可数）的VsCode IDE。
